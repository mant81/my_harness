## 1. 정의 — 무엇이 다른가

**토론 패턴**: 2명+ 에이전트가 같은 질문에 **서로 다른 입장**을 맡아 **상대 주장을 보고 반박**하며 라운드를 돌고, **심판(오케스트레이터)이 수렴 판정**한다.

기존 패턴과의 경계(이게 핵심 — 안 그러면 복합으로 충분):
| 패턴 | 에이전트 관계 | 상호 참조 |
|------|--------------|----------|
| 팬아웃/팬인 | 독립 병렬 | **서로 안 봄** |
| 생성-검증 | 비대칭(생성↔검증) | 검증→생성 단방향 |
| 전문가 풀 | 택일 라우팅 | 하나만 실행 |
| **토론(신규)** | **대칭 대립** | **서로 보고 반박(양방향·라운드)** |

→ 구별 자질 = **교차 반박(cross-rebuttal) + 라운드 + 심판 판정**. 팬아웃 에이전트는 서로를 절대 안 본다. 그 차이가 패턴을 가른다.

## 2. 토폴로지

```
        ┌─ [논객A: 입장 X] ─┐
[쟁점] ─┤    ↕ 반박(R라운드)  ├─→ [심판] → 판정
        └─ [논객B: 입장 Y] ─┘
              (필요시 논객C: 입장 Z)
```

- **논객(debater) 2~3**: 각자 **명시적으로 다른 입장** 배정(찬/반, 또는 접근A/B/C).
- **심판(adjudicator) = 오케스트레이터**: 논객 겸직 금지(이해상충). 최종 판정 **비위임**(external-review와 동일 권위 원칙).

## 3. 적합/부적합

**적합:**
- 정답이 하나로 안 떨어지고 **트레이드오프가 큰 의사결정** — 아키텍처 선택, 기술 채택, 투자 찬반, 정책 설계.
- 다관점이 **충돌**해야 품질이 오르는 경우(레드팀 vs 블루팀 보안, 낙관 vs 비관 리스크 평가).

**부적합(안티패턴 — 쓰지 마라):**
- **증거가 한쪽으로 명백** → 토론 강제 = 거짓 균형(false balance)·토큰 낭비. 심판이 **조기 종료** 허용.
- 단순 병렬 수집으로 충분 → 팬아웃이 맞다(반박 불필요).
- 객관적 검증기준이 있는 품질검수 → 생성-검증이 맞다.

## 4. 라운드 제어 (external-review-loop 재사용)

신규 발명 아님 — 기존 루프 제어를 **그대로 재사용**:
```
round=1
while True:
  각 논객이 상대 직전 주장 반박(round==1은 입장 개진)
  심판: 새 논거 있나? 없으면 dry_streak+1
  if 입장 수렴 or dry_streak>=K: break(converged)
  if round>=MAX_ROUNDS(기본 3): break(max-rounds, 미수렴 명시)
  round++
```
- 종료 라벨: `converged`(수렴) / `max-rounds`(교착 — 심판 강제판정 + **소수의견 기록**) / `early-clear`(한쪽 명백, 조기종료).
- 판정 어휘(external-review 판정표 변형): **채택 / 절충(부분) / 보류(이월) / 기각** + 근거.

## 5. 팀 모드 적합성

**반드시 에이전트 팀.** 교차 반박이 패턴의 본질 — `SendMessage`로 논객 간 실시간 반박, 심판은 모니터링. 서브 에이전트(격리)로는 반박이 성립 안 함(서로 못 봄 = 팬아웃으로 퇴화).

## 6. 그루핑크 방지 (external-review 교훈 이식)

핵심 리스크: 논객이 **같은 엔진**이면 같은 맹점 공유 → 입장만 다르고 *진짜 대립 아님*(가짜 토론).
- 1차 방어: **입장을 강하게 배정**(역할 프롬프트로 "너는 X를 끝까지 변호") — 같은 엔진이라도 상반된 논거 탐색 강제.
- 강한 보장이 필요하면(중대 의사결정): 논객을 **다른 엔진**으로(external-review처럼 codex·agy 활용) — 진짜 독립 관점. 단 비용↑.
- 심판은 **합의=정답 금지** 원칙 유지(external-review와 동일) — 두 논객이 같은 결론도 상관오류일 수 있음, 실근거 대조로 확정.

## 7. external-review-loop와의 관계

토론 = **일반형**, external-review = **특수형**. external-review는 "리뷰어 vs 산출물"의 비대칭 검증이지만 판정 엔진(확인/부분/이월/기각·라운드·loop-until-dry)은 동일 골격. → 토론 패턴은 그 골격을 **의사결정 일반**으로 확장. 코드 추가 최소(판정/루프 재사용).

## 8. 비용·리스크 게이트

토론 = N논객 × R라운드 = **토큰 곱셈**. 슬림 원칙상 **기본 OFF**:
- 경량/표준 도메인 → 토론 안 씀(팬아웃·생성-검증으로 족함).
- **중대 의사결정만** 토론 켬(리스크 등급 연동, Phase 5-6과 동일 철학).
- 논객 2명·MAX_ROUNDS 2 기본(3명·3라운드는 정말 갈릴 때만).

## 9. 복합 패턴 추가

| 복합 | 구성 | 예시 |
|------|------|------|
| **토론 → 파이프라인** | 토론으로 설계 결정 → 파이프라인으로 구현 | 아키텍처 찬반 토론 후 채택안 구현 |
| **팬아웃 → 토론** | 병렬 수집 후 상충 지점만 토론 | 다출처 리서치 → 모순 데이터 대립 검증 |

## 11. 자체검토 — 선반영 약점

1. **[치명 후보] 같은 엔진 = 가짜 토론** — 오케스트레이터(Claude)가 spawn한 논객은 전부 Claude → 같은 맹점·같은 사전확률. "토론"이 *한 모델이 자기와 논쟁*하는 연극이 되어 편한 중간값으로 수렴하기 쉽다(진짜 대립 아님). 입장배정(§6)은 약한 방어. **진짜 대립은 엔진 다양성이 사실상 필수** → 그러면 비용↑ + external-review-loop(다엔진 교차검증)와 **거의 동형**이 됨. ⇒ 별도 패턴 자격이 흔들리는 핵심 쟁점(§13-1·§13-2와 직결).
2. **구별 자질이 교차반박 하나에 전적 의존** — 그게 없으면 토론은 그냥 "팬아웃+생성-검증 복합"이다(§1 표가 유일한 방어선). 판정 시 "논객이 라운드 간 서로 반박하는가?"가 Yes여야만 토론, No면 패턴 호출 금지.
3. **SKILL 500/500 = 여유 0(실측)** — 7번째 패턴 1줄 추가하려면 **반드시 다른 줄을 압축/이동**해야 함. 무게 순증 불가. §10-3대로 미확보 시 SKILL엔 포인터만, 패턴 본문은 references로.
4. **더 싼 제3안 존재** — *새 패턴 추가* 대신 (a) 복합패턴 표에 "팬아웃+생성-검증 = 토론형" 1행만 추가, 또는 (b) external-review-loop 문서에 "이 판정엔진은 *의사결정 토론*에도 쓸 수 있다" 1절 추가. 둘 다 신규 패턴보다 가벼움. 신규 1급 패턴이 이 둘보다 값어치 있는지가 채택 관문.

## 12. 결정 기준

**채택(AND):** 6패턴과 경계 명확(§1 교차반박 자질) + external-review 재사용으로 신규코드 최소 + 슬림(기본 OFF·리스크 게이트) + SKILL 500줄 캡 유지 + 외부감사 수렴.
**기각(OR):** 복합(팬아웃+생성-검증)으로 충분 판명 / 그루핑크로 "가짜 토론" 불가피 / SKILL 캡 못 지킴(무게 초과).
