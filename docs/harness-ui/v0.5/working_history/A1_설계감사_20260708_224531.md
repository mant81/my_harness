## 2. 검증 결과 (CLI 실측 — 맹신 금지)
- `claude --help`: `--allowedTools, --allowed-tools <tools...>` **존재**. `--tools` 없음.
  → agy#3(High "`--allowedTools` 없음→`--tools`, 전 실행 크래시") **기각**. 맹신 시 오히려 버그 주입.
  단 `<tools...>` variadic → 콤마 1문자열 아닌 배열 전달이 정확(Med 정제).
- `codex exec --help`: `--json`·`--ignore-user-config`·`-s/--sandbox`·`-o(--output-last-message)`·`--output-schema` **전부 실존**. 문서 정확.
- `claude -p [prompt]` positional + `--` end-of-options: clap 표준, 저위험. `Task:\n` prefix로 command명(login/mcp…) 충돌 회피 = 타당.

## 3. 판정 — MUST-FIX 6건 (High, codex+agy 수렴)
1. **크로스플랫폼 supervisor 붕괴** (codex#1-3·agy#2) — reconcile POSIX 전용. `kill(-pgid,0)`·음수PID 시그널·process group·타 프로세스 `environ`(`/proc`=Linux·`ps eww`=macOS·Windows 전무)·portable birth-time 없음. Windows 수용 불가.
2. **stdio pipe가 detached child 죽임** (agy#1) — `["ignore","pipe","pipe"]`+detached → supervisor 사망 시 pipe read-end 닫힘 → child EPIPE 사망 → orphan reconcile 전제 붕괴.
3. **닭-달걀 생산자 부재** (codex#4·agy#4) — UI가 읽는 `_workspace/runs/**` 스키마를 현 오케스트레이터가 안 씀. 수용기준은 실 스모크가 그 파일 생성 요구.
4. **범위 vs 수용 모순** (codex#5·agy#9) — 구현순서는 dry-run(step13)까지, 수용기준은 실행·cancel·kill·재시작 reconcile 요구. supervisor 단계 누락.
5. **token stdout scrollback / 멀티유저** (codex#8·agy#5) — 접속 URL 터미널 출력 → 히스토리 유출.
6. **realpath TOCTOU + `_workspace` 심링크 탈출** (codex#9·agy#7).

### SHOULD (Med/Low)
events.jsonl 크래시복구·seq·fsync(codex#10,11) / 스키마 빈약(schemaVersion·종료타임스탬프·cancelRequestedAt·exitSignal, manifest가 targets·permissionMode·model 누락 — codex#12·agy#8) / 커서 페이지네이션 의미(codex#13) / dev-proxy Host·`::1`(codex#14) / artifact 대용량·바이너리·파일명 injection(codex#15) / `--allowedTools` variadic 포맷(codex#7) / drift 스키마 정의(codex#17) / 구현순서에 Run Supervisor 삽입(agy#9).
