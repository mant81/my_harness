## 2. R6 판정
**Blocking:** ①Windows execFile — `start`=cmd 빌트인·`npm`=npm.cmd → ENOENT(codex#1·agy#2) ②토큰 URL 모순 — §5b 쿼리토큰 금지인데 §5c 토큰 URL + 멱등 재오픈이 single-use 토큰 소진→접근불가(codex#2·agy#1).
**High:** npm lifecycle RCE(consent가 postinstall 임의코드 가림, 변조 lock→RCE — codex#3,4·agy#3) · 앱소스 닭달걀(install-time 코드 불가한데 harness-ui/ 없으면 A30 무단계 — codex#5) · pidfile 스푸핑→DoS/오리다이렉트(codex#6·agy#4).
**Med/Low:** npm 별도 확인(codex#7) · cwd realpath 경계(codex#8) · PATH hijack→process.execPath(agy#5) · URL 구성(codex#9).

## 3. r6 경화 (적용)
- Windows: `npm.cmd` 해소, 브라우저 `cmd /d /s /c start "" <url>`, node=`process.execPath`, npm 절대경로(PATH hijack 차단). A30w 신설.
- 토큰: 멱등 재오픈 시 **실행 supervisor에 새 single-use bootstrap 요청**, 토큰 URL/쿼리/로그 미노출. A34 신설(§5b 정합).
- npm: 기본 **`npm ci --ignore-scripts`**(lifecycle 차단) + **lock 해시 템플릿 대조 fail-closed**, postinstall 필수 시 2차 RCE 경고 동의. A33 강화.
- 닭달걀: 첫 실행 **템플릿 스캐폴드**(realpath 프로젝트루트 하위·심링크 거부) 후 설치. A30 절차 1 신설.
- pidfile: raw 신뢰 폐기 → **§4-A 서명 레지스트리 + 헬스체크**로 실행중 판정. A31 갱신.
- cwd=harness-ui realpath 고정·최소 env·package root/lock 심링크 거부.
