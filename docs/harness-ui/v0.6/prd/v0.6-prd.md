## 1. 배경 & 방향 재편

v0.5는 **fire-and-observe** 모델(정본 §1·§3). 초기 v0.6 초안은 세 마찰(대화형·에이전트 1on1·projectRoot 고정)을 신기능으로 잡았으나, **영향평가 결과 방향을 "관측 심화 중심"으로 재편**했다.

### 핵심 전환: 대화형 채팅(구 F1) 폐기
- **판정:** v0.6 **비목표 → v0.7 이월**(완전 삭제 아님·결정 기록으로 보존, 설계서 §F1-폐기결정).
- **근거:** ① 이 앱 목적 = **관측·통제 패널**(fire-and-observe), 채팅 아님. ② 대화는 터미널(Claude Code/Codex)이 이미 **우월·자연스럽다**. ③ Web 채팅은 보안비용 큼(follow-up 세션 resume·codex resume 실CLI **미검증**·stdin/PTY는 정본 §3 EPIPE-safety 위반). ④ 차별화 약함.

### v0.6 새 방향: CLI가 못 하는 것 = 차별화
Web 패널이 터미널보다 나은 지점 = **교차-run 관측**. 여기에 집중한다(전부 **읽기전용**):
1. **Runs 이력 고급 조회·필터·검색(F4).**
2. **문서/artifact 뷰어(F5).**
3. **관측성 계층 B 흡수(F6).**
유지(축소): **F2 에이전트 프리필 New Run**(대화 아님)·**F3 projectRoot 편집**(멀티프로젝트 관찰 정합).
추가(사용자 요청): **F7 에이전트/스킬 정의 편집기** — "스킬과 에이전트를 직접 수정." 이는 **읽기전용 불변식(I8)을 깨는 첫 mutating 기능 = 최대 공격면**이라 경화(화이트리스트·무결성·원자·동시성·스코프 게이트)가 핵심. 방어 확실히 된 뒤 **마지막 배포**.
추가(사용자 요청): **F8 평가 대시보드 + 자기개선 제안 + 평가지표 관리(9번째 화면 Eval)** — "평가 결과 확인 + 평가로 지속 개선 + 지표 관리." 하네스 self-eval(`loop_scorecard`)을 UI로 잇되 **교리 비협상**: `alignment_score`≠품질·**자동 적용 절대 금지(제안+사람 승인만)**·하드 게이트(minAdjudicated≥30·rollingN≥10)·단계 3/4(자동 환류)는 실험.

## 2. 대상 사용자
- **로컬 단일 사용자**(정본 위협모델 유지 — 127.0.0.1 dev-tool, 다중 테넌트 아님). 자기 머신에서 여러 하네스 프로젝트를 오가며 에이전트 팀을 돌리고 **이력·산출물·효과를 관찰**하는 개발자.
- 신뢰 경계는 "로컬 브라우저 ↔ 로컬 서버"지만, **F5(문서/artifact 열람)·F3(경로 권한)**은 로컬이라도 **경로탈출**이라는 새 공격면을 열므로 "로컬이라 괜찮다" 처리 금지(에이전트 원칙).

## 3. 기능별 문제 정의 · 성공지표

### F4 — Runs 이력 고급 조회·필터·검색 (새 주축 · 읽기전용)
- **문제:** run이 쌓이면 CLI/파일로는 "실패한 codex run만", "지난 30분", "에이전트별", "이 문구 포함"을 교차 조회하기 어렵다. 현 `GET /api/runs`는 전체 나열뿐(status만 read).
- **성공지표:**
  - S1: 상태·런타임·기간·mode·에이전트로 **교차-run 필터** + 텍스트 검색 + 정렬 + 페이지네이션(측정: 필터 조합 e2e·결과 정확). *(conversationId 필터는 v0.6 없음 — F1 폐기로 필드 부재.)*
  - S2: **완전 읽기전용** — `events.jsonl`·`status`·`manifest` 스캔만, 파일 변경 0(측정: I8 회귀 0).
  - S3: 대용량 안전 — ReDoS·OOM 없음(측정: 악성 쿼리·대량 run 스위트 통과).

### F5 — 문서/artifact 뷰어 (읽기전용 · 경로탈출 방어 중대)
- **문제:** `docs/**`(설계서·PRD·working_history 결과서)·run artifacts를 화면에서 못 본다(파일 열어야 함). 그러나 임의 열람 = **경로탈출**(`/etc`·`~/.ssh`·`.env`·`.git`) 정면 공격면.
- **성공지표:**
  - S4: docs·run artifacts를 브라우징·열람(마크다운/코드/텍스트 렌더)(측정: 정상 열람 e2e).
  - S5: 경로탈출 **fail-closed** — 화이트루트 밖·심링크·민감파일·바이너리 전건 거부(측정: 위협 스위트 전건 거부).

### F6 — 관측성 계층 B 흡수 (읽기전용 집계 · measured/estimated)
- **문제:** 하네스가 얼마나 효과적으로 쓰이는지(에이전트/스킬 사용·토큰·미사용 dead weight·리뷰 수렴) 안 보인다. events는 귀속 필드(`agent/skill/usage`)를 v0.5에 선반영했으나 소비 UI가 없다.
- **성공지표:**
  - S6: agent/skill별 사용·토큰·미사용 목록·효과성 집계 표시(측정: metrics API + Overview/Agents/Skills 편입).
  - S7: **과대표시 금지** — measured/estimated 신뢰등급 배지, 추정을 정확값처럼 표시 0(측정: estimated 라벨 회귀).

### F2 — 에이전트 프리필 New Run (대화 아님 · 축소 유지)
- **문제:** 특정 에이전트에게 위임하려면 Build에서 domain·tools를 수기 재구성.
- **성공지표:**
  - S8: Agents "이 에이전트에게 요청(New Run)" → 정의 프리필 편집폼 → 기존 `POST /api/runs`(측정: 프리필 필드 ≥3·단일 경로 재사용).
  - S9: allowedTools는 **`U⊆D`(에이전트 선언 도구 내)·조용한 드롭 금지**(UI는 D 선언 도구 체크박스로만·D 밖 요청은 400 반려·A100)·**제출 시점 정의 재조회·D 재도출(템플릿 D 신뢰 금지·정의 삭제/변경 시 `409 agent-definition-changed`·천장우회 차단·R4-#1)**(측정: 미확인 tool 미포함·D 밖 400·삭제/변경 에이전트 제출 409 단위테스트).

### F3 — Settings projectRoot 편집 (보안 자기완결 · 위협모델 강화)
- **문제:** 프로젝트 전환에 서버 재기동 필요(env 고정). 화면 편집 = 멀티프로젝트 관찰 정합, 그러나 **임의 경로 = 경로탈출**(중대).
- **성공지표:**
  - S10: 허용 경계 안 다른 하네스로 전환(측정: 정상 전환 e2e).
  - S11: 경계 밖(시스템/민감·심링크·마커부재·TOCTOU 스왑)은 **fail-closed 400**(측정: 위협 스위트 전건 거부).
  - S12: 변경은 **감사 가능·가역**(config 지속·롤백·env override 최우선).

### F7 — 에이전트/스킬 정의 편집기 (첫 mutating · 최대 공격면)
- **문제:** 사용자가 에이전트/스킬 정의를 화면에서 직접 못 고친다(파일 열어 수정해야 함). 요청: "스킬과 에이전트를 직접 수정." 그러나 이는 **읽기전용 원칙(I8)을 깨는 첫 쓰기** = 경로탈출·손상 정의·lost-update·무단 실행오염의 정면 공격면.
- **성공지표:**
  - S13: `.claude/agents/*.md`·`.claude/skills/**/SKILL.md`를 조회→편집→검증→저장(측정: 정상 편집·저장 e2e·**직렬화본을 실 런타임 리더로 파싱한 differential 게이트 통과**).
  - S14: 쓰기 안전 **fail-closed** — 경로탈출·심링크 write·`.claude` 밖·손상 정의(frontmatter/필수필드/name리네임)·stale write·게이트 off는 전건 거부(측정: F7 위협 스위트 전건 거부).
  - S15: **편집≠실행** — 저장은 파일 기록만·실행 트리거 안 함(측정: 저장이 run 생성 0)·`.bak` 롤백·낙관적 동시성 409.

### F8 — 평가 대시보드 + 자기개선 제안 + 평가지표 관리 (Eval 화면)
- **문제:** 하네스 self-eval(scorecard·summary.jsonl)이 파일로만 존재 — 평가 결과를 못 보고, 악화 추세를 놓치고, 평가지표를 조정할 UI가 없다. 그러나 자기채점 UI는 **Goodhart(자기 점수 올리려 기준 왜곡)·플래핑·alignment을 품질로 오인**하는 위험.
- **성공지표:**
  - S22 (결과 확인·위조 방어·회전 무브릭): scorecard 추세(측정: 읽기전용·경로방어·**ingest=서버 재도출-후-서명(원장서 집계 재계산·`_workspace` precomputed 신뢰 안 함·자기일관 위조 aggregate 격리·R7)→이후 무결성=rollup digest(회전 후 window 브릭 0)**·500 DoS 0·**추세=신뢰 rollup 소싱이라 `_workspace` 가짜 추세 UI 기만 0**). **`alignment_score`를 품질/정밀도로 표시 0·자기채점=약증거→사람 승인 backstop(A103·A107)**.
  - S23 (제안+승인·자동 금지): 악화 트리거→근거 제안(XSS 0)→**서명 envelope(config 결속·가변 rollup-head 미결속)+durable nonce→F7 payload/config 해시 일치+A106 현재 rollup 재평가·독립 저장**(측정: **자동 적용 0·Stage 4 쓰기 0**·payload 은밀 교체 0·**체인 링크·seq·head 불일치·절단 0**·**키 회전 후 과거 엔트리 브릭 0·활성 window `_workspace` 사후 변조 게이트 영향 0(게이트=rollup만·gate-time `_workspace` 재읽기 없음·R6)**·**대기 중 append돼도 정당 제안 도달**·stale config 409·flood/29·9 발화 0·단계<3 비활성·tools/skills). *(state_home 조율 rollback=게임오버·out-of-scope.)*
  - S24 (지표관리 안전·조작성): 채택 단계(쓰기 1-3)·임계·가중치 저장(측정: **per-leaf 복구·형제 손상이 엄격값 소거 0·리셋값 floor(30/10/3) 밑 0**·부재/손상→안전기본값·**입력 옆 최소 30/10/3 상시 표시·floor 미만 인라인 거부(silent clamp 0)·effective diff·적용값 피드백**·**확인=단계 3 전환+지표/정규화 변경·Stage 4=잠금 사유만**·UX-R1-#4).

- **F4:** `GET /api/runs` 필터/검색/정렬/페이지 확장(manifest 병합 델타·Zod·리터럴 검색·**이름 열거→`fs.stat` 시간 정렬→상위 N 내용 바운드 read→전역 정렬**·runId 형식 무의존), Runs 목록 필터바.
