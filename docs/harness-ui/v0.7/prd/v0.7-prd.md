# Harness UI v0.7 — PRD (헤드라인: F-CLI 세션 로그 관측 · CLI 실행 가시화)

> 상태: **기획 초안 · 미검증(외부감사 대기).** 정본 근거 = `docs/harness-ui/v0.6/design/design-v0.6.md`(A47-A112·불변식 I1-I8·공용 경화 바운드-리더·measured/estimated 라벨)·`docs/harness-ui/v0.5/design/design-v0.5-final.md`(코어 §1-9·A1-A46·로컬 단일사용자 위협모델).

## 1. 배경 — v0.6의 구조적 한계
v0.6까지 UI는 **자기(supervisor)가 fire-and-observe로 실행한 run만** 관측한다(정본 §3·I1). 그러나 실사용의 상당 부분은 **터미널에서 직접 실행한 CLI**(`claude`·`codex`)다 — 그건 UI에 **전혀 안 보인다**. 결과:
- **History가 반쪽:** supervisor run만 나오고 터미널 세션은 누락 → "내가 이 프로젝트에서 뭘 돌렸나"의 절반만.
- **활용도 통계가 반쪽:** v0.6 F6은 supervisor events만 집계. 터미널에서 쓴 에이전트/스킬은 미반영 → v0.6이 관측성 근거 약해 활용도(계층 B 일부)를 축소.

**v0.7이 이걸 연다:** 터미널 CLI가 남기는 **세션 트랜스크립트를 읽어(read-only)** UI에 표시. 단, CLI 트랜스크립트는 **벤더 소유 포맷 + 사용자 전 프로젝트 이력(프롬프트·코드·비밀)**이라 **프라이버시·벤더 취약성**이 최우선 설계 쟁점.

## 2. 대상 사용자
- **로컬 단일 사용자**(정본 위협모델 유지·127.0.0.1 dev-tool). 자기 머신에서 supervisor run과 **터미널 CLI를 병행**하며 "이 프로젝트에서의 전체 활동"을 한 곳에서 보고 싶은 개발자.
- 신뢰 경계 = "로컬 브라우저 ↔ 로컬 서버". 단 F-CLI는 **`~/.claude` 라는 projectRoot 밖 read root**를 새로 여므로(v0.5 "projectRoot 하위만" 경계 확장) **명시 옵트인·스코프·경로방어**를 못박는다.

## 3. F-CLI — CLI 세션 로그 관측
- **문제:** 터미널 `claude`/`codex` 실행이 UI에 안 보여 History·활용도가 실사용 전체를 반영 못 함.
- **접근(읽기전용):** `~/.claude/projects/{현 projectRoot 인코딩}/*.jsonl`(Claude Code 세션 트랜스크립트·NDJSON) + codex 상당 로그(`~/.codex/**`·포맷 미확인)를 **읽어** 표시. **supervisor run과 구분**·필터.
- **소비처:**
  - (a) **History에 "CLI 세션" 소스 추가** — supervisor run과 **별 소스로 구분 표시**·필터(혼합 금지).
  - (b) **활용도 실데이터 재도입 검토** — v0.6서 축소한 활용도(에이전트/스킬 실사용)를 **CLI 실데이터 위에** 부활(best-effort 귀속).
- **성공지표:**
  - S1 (가시화): 현 projectRoot의 터미널 CLI 세션이 History에 **supervisor run과 구분되어** 표시(측정: CLI 세션 목록 e2e·소스 배지 구분).
  - S2 (프라이버시·비협상): **명시 옵트인 없이는 `~/.claude` 미열람**·기본 **현 projectRoot 세션만**·전역/타 프로젝트 열람 기본 차단·**UI 밖 송출 0**(측정: 옵트인 off 시 발견 0·전역 열람 시도 거부·외부 요청 0).
  - S3 (벤더 취약 견딤): 트랜스크립트 스키마가 바뀌거나 깨져도 **graceful**(전체 안 죽음)·**"CLI 추정" best-effort 라벨**(measured 오표시 0)(측정: 손상/미지 스키마 fixture graceful·라벨 회귀).
  - S4 (귀속 정직): Task(subagent_type)·Skill 툴콜에서 에이전트/스킬 **추정** 매핑·불확실은 "추정/미귀속"(측정: 과대표시 0·estimated 규율 준용).

## 4. 범위 (v0.7 포함 · F-CLI 1순위)
- **F-CLI(상세설계·M14):** 소스 발견(현 projectRoot 인코딩 스코프)·fail-soft 파싱·best-effort 귀속·**공용 경화 바운드-리더(v0.6 재사용·앵커=`~/.claude/projects/{enc}`)**·프라이버시 위협모델(옵트인·스코프·경로방어·송출 0)·신뢰 라벨·History "CLI 세션" 소스·`GET /api/cli-sessions*`(읽기전용·side-effect 0).

## 6. 비목표 (F-CLI 범위 밖 · 명시)
- **전역 `~/.claude/projects` 브라우저**(전 프로젝트 이력 열람) — 기본 차단·현 projectRoot 스코프만. 타 프로젝트 열람은 별도 명시 확장(신중·기본 off).
- **CLI 세션 쓰기·수정·재개**(read-only 관측만·supervisor 대화형 아님·구 F1 PTY 폐기 유지).
- **트랜스크립트 원문 전량 표시**(민감정보) — 요약/메타 기본·원문은 명시 확장 열람.
- **CLI usage를 measured로 표시** — 벤더 파싱 best-effort는 "CLI 추정" 별 등급(과대표시 금지).
- **외부 송출·텔레메트리**(UI 밖 0).

## 7. 우선순위
1. **F-CLI 소스 발견·경화 리더·옵트인·스코프**(프라이버시 게이트 먼저 — 잘못 열면 사용자 전 이력 노출).
2. **fail-soft 파싱·신뢰 라벨**(벤더 취약 견딤·정직 표시).
3. **History "CLI 세션" 소스 구분**(엔티티 혼합 금지).
4. **best-effort 귀속·활용도 재도입 검토**(F-CLI 위에·estimated 규율).

## 다음 단계 참조
- **미해결(assumption·설계서 §assumption 격리):**
  1. **Claude 트랜스크립트 스키마**(NDJSON·`type`/`sessionId`/`leafUuid` 실측 확인) — 벤더 소유·버전 변동 → fail-soft. Task/Skill 툴콜 구조로 귀속 가능한지 실측 확정 필요.
  2. **디렉토리 인코딩 규칙**(projectRoot `/`→`-` 매핑) — 현 projectRoot→dir 매핑 정확성·leading dash·특수문자·중첩 경로 충돌.
  3. **codex 로그 포맷**(`~/.codex/sessions|log` 존재 확인·포맷 미확인) — assumption·미지원 graceful.
  4. **활용도 재도입 범위** — CLI-추정 등급을 활용도에 얼마나 반영할지(과대표시 위험 vs 실사용 가치).
- **핵심 결정 & 이유:**
  - **F-CLI = 읽기전용 관측만** — v0.5/v0.6 불변식(I1-I8·읽기전용 원칙) 준수·CLI 쓰기/재개/대화형 아님.
  - **프라이버시 최우선** — `~/.claude` = projectRoot 밖 새 read root라 명시 옵트인·현 projectRoot 스코프·경로방어·송출 0을 게이트로 먼저.
  - **벤더 포맷은 신뢰 못 함** — fail-soft·"CLI 추정" 별 등급(measured 아님)·스키마 깨져도 graceful.
  - **엔티티 구분** — CLI 세션 ≠ supervisor run·History 별 소스(혼합 금지).
